Data Manipulation Language (DML)

Data Manipulation Language (DML) is a subset of SQL (Structured Query Language) used to manage and manipulate data within database objects such as tables. DML commands are essential for querying, inserting, updating, and deleting data stored in the database. These commands are the primary means of interacting with and managing the actual data.

Key DML Commands

SELECT:

Used to retrieve data from one or more tables.
Example:
Selecting all columns from the "employees" table..

INSERT:

Used to add new rows of data to a table.
Examples:
Inserting a new row into the "employees" table with specific values for each column.

UPDATE:

Used to modify existing data within a table.
Example:
Updating the hire date of an employee in the "employees" table.

DELETE:

Used to remove rows of data from a table.
Example:
Deleting a specific employee's record from the "employees" table based on their employee ID.

Characteristics of DML Commands

Transactional:

DML commands can be rolled back or committed, allowing for greater control over data manipulation and consistency.

Data Handling:

These commands are used to insert, update, delete, and retrieve data, making them crucial for day-to-day database operations.

Conditional operation:

DML commands often include WHERE clauses to specify conditions for data manipulation, providing precise control over which rows are affected.
