ASSIGNMENTS

Data Security:

Level 1 - Physical Security:

Concerned with securing physical access to data storage devices, servers, and data centers.

Level 2 - Human Security:

Involves policies and procedures to ensure that personnel handling data follow security protocols and practices.

Level 3 - Operating System (OS) Security:

Measures to secure the operating systems of computers and servers against unauthorized access and attacks.

Level 4 - Network Security:

Techniques and technologies used to protect data during transmission over networks, including encryption and firewalls.

Level 5 - Application Security:

Securing software applications and APIs from vulnerabilities and unauthorized access.

Data Integrity Constraints:

Entity Integrity Constraint:

Ensures that no primary key attribute in a table is null, thereby maintaining the uniqueness and integrity of records.

Domain Integrity Constraint:

Enforces valid data types, ranges, or formats for data stored in a database to prevent invalid data entry.

Referential Integrity Constraint:

Ensures relationships between tables remain consistent by enforcing foreign key constraints, preventing actions that would destroy links between tables.

Anomalies:

Insertion Anomaly:

Occurs when certain attributes cannot be added to the database without the presence of other attributes.

Deletion Anomaly:

Happens when deleting certain data also unintentionally removes other data that should remain in the database.

Update Anomaly:

Arises when updating data results in inconsistencies due to failure to update all occurrences of the data.

These categories provide a structured way to understand and address different aspects of data management and security, ensuring that data remains secure, accurate, and consistent throughout its lifecycle.
