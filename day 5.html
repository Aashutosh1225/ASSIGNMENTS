Data Definition Language (DDL)

Data Definition Language (DDL) is a subset of SQL (Structured Query Language) used to define and manage database schema. DDL commands are essential for creating, modifying, and deleting database structures, such as tables, indexes, and views. Unlike DML (Data Manipulation Language), which deals with the manipulation of data within these structures, DDL focuses on the schema or structure itself.

Key DDL Commands

CREATE:

Used to create database objects such as tables, indexes, views, and databases.
Example:
Creating an "employees" table with columns for employee ID, first name, last name, and hire date.

ALTER:

Used to modify an existing database object.
Examples:
Adding a new column to a table. Modifying the datatype of an existing column. Dropping a column from a table.

DROP:

Used to delete database objects.
Example:
Dropping the "employees" table permanently removes the table and its data.

TRUNCATE:

Used to remove all records from a table but keeps the table structure for future use.
Example:
Truncating the "employees" table removes all rows but retains the table for future data insertion.

RENAME:

Used to rename database objects.
Example:
Renaming the "employees" table to "staff".

COMMENT:

Used to add comments to the data dictionary.
Examples:
Adding a comment to a table. Adding a comment to a column in a table.

Characteristics of DDL Commands

Autocommit:

DDL commands execute with an implicit commit, meaning changes are automatically saved and cannot be rolled back.

Schema Management:

These commands are fundamental for managing the schema of the database, ensuring that structures are defined before data manipulation.

Integrity Constraints:

commands can define constraints such as PRIMARY KEY, FOREIGN KEY, UNIQUE, CHECK, and NOT NULL to enforce data integrity and consistency
